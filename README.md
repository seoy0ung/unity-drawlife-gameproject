# ✏️ DRAWLIFE (게임 프로젝트)

> 2021년에 제작했던 4단계 생애 주기 기반 시뮬레이션 게임 프로젝트입니다.
> 유저의 선택과 플레이에 따라 유아기부터 성인기까지 성장하며 다양한 엔딩을 수집할 수 있습니다.

## 🎮 게임 소개
**DRAWLIFE**는 플레이어가 주인공이 되어 4단계의 생애 주기(유아기 ➡️ 아동기 ➡️ 학창시절 ➡️ 성인기)를 거치며 일어나는 다양한 이벤트와 미니게임을 통해 성장하는 게임입니다. 플레이어의 선택(아이템, 직업 선택 요소 등)과 게이지 달성도에 따라 결과(직업/엔딩)가 달라지는 시뮬레이션 요소를 담고 있습니다.

## 🛠 기술 스택
- **Engine**: Unity Engine (2D)
- **Language**: C#
- **Version Control**: Git / GitHub

## ✨ 주요 기능 및 로직
- **생애 주기 상태 관리 (State Management)**
  - `nstate` 상태 변수를 기반으로 4단계의 생애 주기를 분리하여 제어.
  - 각 단계에 맞춰 배경 이미지, 캐릭터의 스프라이트 및 애니메이션(`RuntimeAnimatorController`), 물리 충돌(Collider) 영역을 동적으로 변환.
- **코루틴을 활용한 비동기 흐름 제어**
  - `IEnumerator` (Coroutine)를 적극 활용하여 시간에 따른 게이지 감소 및 스테이지 진행도 막대 등을 제어.
  - 페이드인/페이드아웃 및 씬 전환 애니메이션과 대기 시간을 자연스럽게 연결.
- **데이터 주도 설계 (Data-Driven Design)**
  - 퀴즈, 아이템 정보, 게이지 변동 수치 등 주요 데이터를 인게임에 하드코딩하지 않음.
  - 외부 데이터(CSV) 파일을 `CSVReader`로 불러와 파싱하고 딕셔너리 형태로 저장하여 유연한 데이터 관리를 구현.
- **세이브 및 컬렉션 시스템**
  - `PlayerPrefs`를 활용한 플레이어 닉네임 저장 및 불러오기.
  - 플레이 결과에 따른 직업별 엔딩 컬렉션 해금 및 로컬 데이터 기록 관리.
- **인터랙티브 미니게임 및 UI/UX**
  - 스테이지 진행 중 발생(QTE, 커맨드 입력 등)하는 미니게임 로직 구현.
  - 사용자 입력 성공/실패 시 즉각적인 시각적 피드백(텍스트, 색상 변화) 제공.

## 🙋‍♀️ 나의 기여 (개발 및 아트)
- **개발 파트**
  - `GameManager`를 중심으로 하는 핵심 게임 사이클 (시간 흐름, 단계 트랜지션, 게이지 로직) 구현.
  - CSV 데이터 파싱 후 인게임 데이터 매핑 및 랜덤 아이템 등장, 퀴즈 출제 로직 구현.
  - 미니게임 판정 로직 및 세이브 데이터 연동 구축.
- **아트 파트**
  - 생애 주기에 따른 남/여 주인공 캐릭터 스프라이트 및 애니메이션 리소스 제작.
  - 스테이지별 배경(Background) 연출 프레임 및 인게임 UI(게이지 바, 미니게임 캔버스, 튜토리얼 등) 디자인 및 오브젝트 배치.
